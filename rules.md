## 1. 🤖 กฎเหล็กสำหรับการทำงานของ AI (AI Core Guidelines)

1. **ลำดับการทำงาน (Prompt History Rule)**:
   - **ต้องเขียน `prompt.md` ก่อนลงมือทำงานตามคำสั่งเสมอ**
   - รูปแบบการบันทึกลง `prompt.md`: ให้แปลคำสั่งเป็นประโยคภาษาอังกฤษ และสลับเอาภาษาอังกฤษ (EN) ไว้ด้านบน ภาษาไทย (TH) ไว้ด้านล่างเสมอ
   - หากคำสั่งมีคำศัพท์ภาษาอังกฤษที่ยาก ให้เขียนคำแปลไว้บรรทัดถัดไปด้านล่างประโยคภาษาไทยทันที โดยใช้รูปแบบ: `- *Vocabulary: english_word = thai_translation*`
   - คำสั่งที่ซ้ำเดิม ไม่ต้องบันทึกซ้ำลงใน `prompt.md`
2. **สิทธิ์การสั่งการ (Auto-Execution)**:
   - AI มีสิทธิ์ในการรันคำสั่งบน Terminal (เช่น git, curl) ได้อัตโนมัติทันทีเพื่อความรวดเร็ว
   - **ห้ามใช้ Browser Tools หรือ Browser Automation** ในการดึงข้อมูล ให้ใช้ `curl` หรือ Python สคริปต์แทน
3. **การซิงค์ข้อมูล (Git Auto-Sync)**:
   - ทุกครั้งที่มีการอัปเดตโค้ดและทดสอบเสร็จแล้ว **ต้องรัน** `git add .`, `git commit -m "..."`, และ `git push origin HEAD` อัตโนมัติทันที
   - ใช้ข้อความ Commit เป็นภาษาไทย หรือภาษาอังกฤษแบบสื่อความหมายชัดเจน
4. **การอัปเดตคู่มือ (Documentation Auto-Update)**:
   - หากมีการเพิ่ม Option, Module ใหม่, หรือเปลี่ยนแปลงการทำงาน ต้องอัปเดตเอกสารคู่มือ (`.md` ต่างๆ เช่น `readme.md`, `style.md`) ให้เป็นปัจจุบันทุกครั้ง
5. **การสนับสนุนการเรียนรู้ภาษาอังกฤษ (English Language Coaching)**:
   - หากผู้ใช้พิมพ์คำสั่งหรือข้อความมาเป็นภาษาอังกฤษ ให้ AI ช่วยแนะนำการใช้คำศัพท์ การจัดวางประโยค (Grammar) ที่ถูกต้อง หรือประโยคที่ฟังดูเป็นธรรมชาติกว่า (Native-like) ต่อท้ายในส่วนของประโยคแปลใน `prompt.md` เสมอ เพื่อให้ผู้ใช้ได้ฝึกฝนและเรียนรู้ภาษาอังกฤษไปในตัว


## 2. 🗂️ กฎการจัดการระบบไฟล์และ Git (File & Git Management Rules)

1. **การตรวจสอบสถานะ Git (Git Status Check)**:
   - **ก่อนการรันคำสั่งใดๆ ที่อาจมีการเปลี่ยนแปลงไฟล์ และหลังการแก้ไขโค้ดเสร็จสิ้น ต้องรันคำสั่ง `git status` ทันที** เพื่อตรวจสอบความเปลี่ยนแปลง
   - วิเคราะห์ความเปลี่ยนแปลงที่เกิดขึ้น และบันทึกประวัติการเปลี่ยนแปลงลงใน `log_update.md` โดยสรุปการเปลี่ยนแปลงที่สำคัญและเหตุผล
2. **การบันทึกประวัติการอัปเดต (Log Update)**:
   - ไฟล์ `log_update.md` ทำหน้าที่เป็นบันทึก (Changelog) ของระบบ
   - **ห้ามใช้ Git Commit เพื่อบันทึกประวัติการอัปเดต** ให้บันทึกเฉพาะในไฟล์ `log_update.md` เท่านั้น
   - ใช้รูปแบบการบันทึกที่ชัดเจน เช่น วันที่, รายการการเปลี่ยนแปลง (Bullet points), และเหตุผล
3. **การจัดการไฟล์ `prompt.md`**:
   - ไฟล์ `prompt.md` เก็บเฉพาะคำสั่งที่ดำเนินการไปแล้วเท่านั้น (Executed Commands)
   - หากคำสั่งซ้ำเดิม หรือเป็นคำสั่งเดิมที่ดำเนินการไปแล้วหลายครั้ง **ห้ามเขียนซ้ำลงในไฟล์ `prompt.md`** ให้เว้นว่างไว้
4. **การจัดการไฟล์ `pending_task.md`**:
   - ไฟล์ `pending_task.md` ทำหน้าที่เป็น **Backlog หรือ Task List** ที่รอการดำเนินการ
   - หากผู้ใช้ระบุ To-do list หรือรายการงานที่ต้องการให้ทำในอนาคต ให้บันทึกไว้ที่นี่
   - **ห้ามลบงานออกจากไฟล์ `pending_task.md` จนกว่างานนั้นจะเสร็จสิ้นสมบูรณ์**
5. **การล้าง Task (Task Clearing)**:
   - เมื่อทำงานใน `pending_task.md` เสร็จสิ้นแล้ว **ต้องลบรายการงานนั้นออกจากไฟล์ทันที** เพื่อไม่ให้เกิดความสับสนว่างานไหนทำแล้วหรือยัง
